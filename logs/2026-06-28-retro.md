# 📅 2026-06-28 | 战役：IntentRouter 路由错乱拦截与 Cosmos 审计闭环

> **第一性原理核心**：
> 1. **公理 2 (概率与确定性隔离)**：在意图分类中，用户的显式提及（如 `@mention`）是 100% 确定性的，绝对不能让概率性的 LLM 路由模型对其进行“越权覆写”；显式声明必须具有最高优先级（P0 级硬阻断）。
> 2. **公理 1 (注意力预算)**：路由器的意图描述必须极简且边界清晰。重叠的发散性描述会混淆大模型的注意力，导致路由判定发生概率性漂移（如“文件齐全查询”误路由至“工作区操作”）。

---

## 🚀 一、 场景演练与实用主义重构对照 (Scenarios & Before-After)

### 📂 【Before ➔ After】IntentRouter @mention 优先权与多层路由修复

*   **痛点 (Antipattern)**：
    *   在开发统一助手时，发现当用户在对话中显式 `@mention` 某个特定的 Sub-Agent 时，系统依然会走后端的声明式意图分类（Probabilistic Router）。AI 在分类时发生失误，经常无视用户的显式指令，强行将请求发散分配给其他模型。
    *   同时，像“文件齐全查询”这种强金融业务日期属性的查询，由于分类 prompt 中的动词重叠，频繁发生路由漂移，误入 `workspace_operation` 智能体（PR #242 故障）。
*   **重构方案 (Pattern)**：
    *   **@mention 确定性提权 (P0 级重构)**：在 `IntentRouter` 的最顶层注入纯代码的硬阻断逻辑。凡是正则检测到 `@mention` 模式，直接跳过一切概率性大模型分类，物理分流手控权（PR #244）。
    *   **意图分类三层拦截重构**：重构 `IntentRouter` 分类 Prompt，剔除发散性的动词定义，强行限定业务边界。在 `valuation_qc` 与 `workspace_operation` 间构建互斥负面清单（Negative Guidelines），成功对齐 VFS 挂载点检测与文件齐全度的路由路径（PR #242）。
*   **实用主义收获**：**“大模型是不可信的，用硬正则和互斥负面清单建立确定性预检，才能死守路由正确率的红线。”**

## ❌ 二、 失败模式深度剖析 (Failure Modes)

### 🛠️ 【反模式】模糊路由规则导致的“路由套娃”与测试遗漏
*   **失败现象**：在设计估值质控路由时，因为分类 prompt 描述中同时包含了 “文件”、“质控” 等发散字眼，导致当输入 `100301 路由审计` 等混合查询时，路由器在两个智能体之间无限重定向，直至单次会话超时抛错。
*   **物理成因**：典型的上下文描述重叠。大模型在两份高度重叠的 “垃圾规则抽屉” 间产生决策分叉（Friction Pattern）。
*   **纠偏行动**：
    *   建立**确定性路由审计制度**。
    *   将 100301 等高频真实查询抽象为**硬编码单元测试用例**，手动执行路由测试并通过 Git 记录提交测试日志（PR #245）。
    *   每次修改路由规则后，必须就地跑通 `pnpm test:router`，确保零路由退行性错误。

## ⚖️ 三、 与 Cursor/Copilot CLI 对比 (Utility博弈)

*   **Claude Code (CLI)**：在执行极速修复-提交-提 PR（PR #241 至 PR #245 五星连珠）的短平快战役中，Claude Code 展现了恐怖的爆发力。一键完成 `@mention` 提权 TS 代码编写后，无需切换界面，顺手利用 CLI 里的 `gh pr create` 命令提交多达五个关键 PR。一站式的终端无人值守极大地消除了“GUI 切换摩擦”。
*   **Cursor (GUI)**：在遇到这种由于高频修改路由 Prompt 带来的“微调-测试-微调”调试飞轮时，Cursor 频繁的弹窗与上下文索引确认会打断开发者的思路，不适合用来做这种多 PR 的极速路由压测审计。

## 🏦 四、 金融场景最佳实践 (Strict Privacy Gotchas)

### 🔒 【第一性原理】IntentRouter Cosmos 审计对齐与操作合规性
*   在金融估值质控平台中，每次 AI Agent 进行的路由跳转（Intent Disambiguation）实质上都等同于一次系统越权操作（Privilege Hand-off）。
*   为此，我们于 2026-06-28 专门向仓库提交了 **Cosmos 意图路由对齐审计报告（PR #243）**。
*   将大模型的推理判定过程（Chain-of-Thought）进行强制的格式化规范，要求 AI 在做出路由跳转决定时，必须同时在 header 输出确定性的 `Reasoning-Token` 审计证据，使其 100% 具备可回溯、可监管的技术底色，优雅解决金融审计合规痛点。
