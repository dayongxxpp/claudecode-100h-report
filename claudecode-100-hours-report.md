## 🧬 第一部分：AI 编程的三大第一性原理公理 (Axiomatic Foundations)

任何 AI 编程工具（无论是 Claude Code CLI、Cursor 还是 Custom Agents），其底层都必须服从以下三大物理定律。全栈 AI 工程师的所有“神操作”都是这三大公理的推论。

### 公理 1：注意力预算定律 (Attention is a Finite Physical Resource)
*   **物理底层**：LLM 的 Transformer 架构在处理上下文时，注意力（Attention）呈平方级或线性损耗。随着 Token 暴涨，检索准确度（Needle in a Haystack）必然下降。
*   **实用主义心智模型**：**垃圾上下文抽屉效应 (Junk Drawer Effect)**。
    *   *反模式*：把整个仓库的 `node_modules`、文件夹结构说明、冗余 linter 规范和“写出干净代码”等废话全塞进 `CLAUDE.md`。这会迅速耗尽 AI 的注意力预算，导致它在重构核心逻辑时“睁眼瞎”。
    *   *第一性原理*：**Path Scoping (路径强限定)**。只给 AI 输入它赖以决策的最小真相源。

### 公理 2：概率性与确定性隔离定律 (Probabilistic vs. Deterministic Isolation)
*   **物理底层**：大语言模型（LLM）本质上是一个**概率预测器 (Probabilistic Engine)**。它输出的每一行代码都是基于概率密度的采样，必然存在失误率（幻觉）。而系统的运行和合规拦截必须是**确定性 (Deterministic)** 的。
*   **实用主义心智模型**：**物理第二防线 (The Double-Phase Shield)**。
    *   *反模式*：在 Prompt 里求 AI *“请千万不要修改其他模块”* 或 *“请千万不要写入沙盒外的绝对路径”*。
    *   *第一性原理*：用确定性的代码写死校验逻辑（如：`path.startsWith(sandbox + path.sep)`），由 Handler 执行绝对物理阻断。把高成本的认知工作交给 AI（概率），把低成本的安全防线交给自己（确定性）。

### 公理 3：约定大于配置减熵定律 (Convention Over Configuration)
*   **物理底层**：AI 的预训练数据中包含了互联网上 20 多年最标准的、遵循行业约定（如 NestJS 标准装饰器、MVC 架构、RESTful 规范）的优质代码。如果你的代码库遵循约定，AI 就能激活其百亿级参数的“无感理解”；反之，如果你标新立异，你就必须花费巨额的 Token 和 Prompt 去向 AI 解释你的“配置”。
*   **实用主义心智模型**：**AI 易感性设计 (AI-Susceptible Codebase)**。
    *   *第一性原理*：写最标准的、遵循社区最广泛公认约定的代码。代码即文档，AI 已经在互联网上学过一万遍了。

## 🔬 第三部分：第一性原理演练场 (Labs & Exercises)

全栈 AI 工程师的成长，唯有通过在沙盒中进行“高频、极限的破坏性测试”来练就肌肉记忆。你可以在本仓库的 [labs/](./labs/) 目录下进行以下三个经典演练：

### 🧪 演练 1：[VFS 边界与物理逃逸自检验证 (Labs/vfs-playground)](./labs/vfs-playground/)
*   **演练目标**：亲自动手写一段 Node.js 代码，模拟一个文件挂载点在接收到 AI 写入包含 `../` 或绝对路径逃逸指令时，如何通过物理 Assert 强制阻断，并观察 AI 的自动报错与回滚反应。
*   **实用主义收获**：理解为什么不能用 Prompt 去求 AI “不要越界”，必须用物理代码卡死边界。

### 🧪 演练 2：[极简注意力预算裁剪挑战 (Labs/prompt-templates)](./labs/prompt-templates/)
*   **演练目标**：将你当前项目臃肿的 `CLAUDE.md` 或系统 Prompt，通过“删除清单规则”强力剪枝。
*   **自检通关条件**：
    1.  文件行数压缩到 **120 行以内**；
    2.  再次让 Claude Code 重构时，单次 Token 开销降低 **70% 以上**；
    3.  AI 逻辑修改的准确率仍能保持在 **100%**。
